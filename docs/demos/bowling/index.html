Bowling for contagion

Companies are pins. Sectors crash together. The overlay knows.

A bowling alley is a generative model for a market. Each pin is a company; companies in a sector stand close together — so when a shock caroms through, they tend to fall together: default contagion. A covariance sees only smooth co-movement and misses the all-or-nothing cascade — but a tail-aware overlay, reading weights as winning probabilities of a crash race, de-duplicates the names that default together. How often each company is the single worst crasher across many random shocks. Sector-mates split that credit, so the overlay underweights tightly-clustered co-defaulters (short bars) and keeps more in the lonelier names (long bars). Bar colour = sector.

Why this is the point

Minimum-variance and Black–Litterman both run on a covariance, and a covariance fit to this world matches a Gaussian copula that cannot represent a whole sector defaulting at once — so they could over-concentrate into a cluster that looks calm on variance and then blows up together. The overlay never inverts a covariance; it races the assets under the real (cascading) simulation, and co-defaulters cannibalise each other's wins, so the win-share of any one contagion cluster is de-duplicated automatically — the bars show that de-duplication.

A caveat worth stating plainly: the bars are the de-duplication mechanism, not a claim of superior returns. In the bowling lab — the same generator run as an out-of-sample horse race across many random markets — the full overlay (taken all the way to a standalone portfolio) actually over-concentrates and takes a worse tail than equal weight or minimum-variance. The lesson the lab teaches: this is a benchmark tilt to apply with a confidence dial, not a free lunch. The tail-sensitive Black–Litterman demo is the same overlay, dialled, on the Dow.
