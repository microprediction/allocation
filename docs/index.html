allocation

Online (streaming) portfolio construction.

scikit-learn / skfolio-compatible estimators (fit / partial_fit / weights_ / predict), on top of a keyed dynamic-universe layer — so they survive reconstituting universes, where assets come and go.

long-only streaming · partial_fit changing universe low turnover skfolio-compatible

Install

pip install allocation
pip install allocation[scale]    # sparse Fiedler (scipy) for large universes
pip install allocation[skfolio]  # skfolio Portfolio interop

Two methods, one engine

Thurstone portfolio

Weights are the winning probabilities of a Thurstonian race among assets. Abilities are calibrated so the race reproduces a benchmark at a reference correlation; the tilt re-runs it under the estimate. A smooth common-seed transport gives low-turnover streaming updates.

Schur-complementary

Peter Cotton's Schur allocation (the gamma coupling interpolating HRP and minimum variance), but with the order from Fiedler seriation instead of agglomerative linkage — so the hierarchy, and the weights, are a smooth function of the covariance.

Batch — scikit-learn / skfolio style

import numpy as np
from allocation import ThurstonePortfolio, SchurComplementary

est = ThurstonePortfolio(calib="market", phi=1.0)
est.fit(returns)              # returns: (n_observations, n_assets)
est.weights_                  # long-only weights on the simplex
est.score(returns)            # Sharpe ratio (skfolio scoring convention)

SchurComplementary(gamma=0.5).fit(returns).weights_

Streaming — river style, over a changing universe

from allocation import StreamingThurstone, StreamingSchur

est = StreamingSchur(gamma=0.5)
for x in stream:                       # x = {asset_id: return}
    est.learn_one(x)                   # assets may enter / leave each step
    weights = est.predict_one()        # {asset_id: weight}

A changing universe is just a changing set of dict keys. Per-asset state is keyed; the smooth transport / Fiedler order carry across updates so turnover tracks genuine change, not churn.

Why a separate package

skfolio is the scikit-learn-native portfolio library, but it is batch: a changing universe is handled by NaN/incomplete-data mechanics rather than by carrying per-asset state across streaming updates. allocation is the online complement — partial_fit, a keyed dynamic universe (in the spirit of precise), and turnover control by construction — while staying API-compatible so each estimator can also be contributed upstream.
